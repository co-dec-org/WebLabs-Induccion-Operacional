# QA Visual

Este documento define los criterios de revisión visual para Web-Labs Inducción Operacional.

## Objetivo

Asegurar que cada página mantenga coherencia visual, funcional y legal-operacional con los mockups oficiales aprobados.

## Mockups de referencia

Los mockups oficiales están ubicados en:

docs/mockups/

Cada página debe compararse visualmente contra su mockup correspondiente.

## Páginas a validar

- Login
- Inicio / Home
- Inducción
- Inducción modo presentación
- Bitácora
- Marco Legal
- Recursos
- Perfil
- Simulaciones deshabilitada
- Supervisión
- Admin

## Checklist visual general

Para cada página se debe revisar:

- layout general;
- proporciones;
- jerarquía visual;
- sidebar;
- topbar;
- footer institucional;
- botón flotante Notas;
- tarjetas;
- botones;
- estados activos;
- estados deshabilitados;
- contraste;
- legibilidad;
- responsive;
- coherencia con tema Boldo;
- coherencia con tema Ámbar.

## Tema Boldo

Debe ser monótono, institucional y sobrio.

No debe usar:

- amarillo;
- dorado;
- naranja;
- ámbar;
- acentos cálidos.

## Tema Ámbar

Puede usar acentos cálidos, pero debe mantener sobriedad institucional y uso formativo.

## Simulaciones

La página Simulaciones debe estar visible en navegación, pero deshabilitada en v0.1.

Debe mostrar el mensaje:

“Funcionalidad no disponible en esta versión. Módulo proyectado para una etapa posterior de entrenamiento operacional.”

## Criterio legal-operacional

La plataforma es formativa. No debe registrar datos reales de víctimas, PSC, domicilios, teléfonos, coordenadas, folios, causas ni antecedentes identificables.

Toda interfaz debe reforzar minimización de datos, finalidad formativa y acceso restringido.
