# Design System

Este documento define los criterios visuales base para Web-Labs Inducción Operacional.

## Objetivo

Mantener una interfaz institucional, clara, accesible y coherente con el uso formativo del Sistema de Monitoreo Telemático.

El sistema visual debe permitir dos temas oficiales:

- Boldo
- Ámbar

## Tema Boldo

Uso principal:

- institucional;
- sobrio;
- monótono;
- bajo ruido visual;
- lectura prolongada;
- páginas oficiales y de consulta.

Criterio visual:

- no usar amarillo, dorado, naranja ni ámbar;
- usar verdes institucionales, blanco, gris verdoso y negro verdoso;
- priorizar contraste, legibilidad y estabilidad visual.

## Tema Ámbar

Uso principal:

- laboratorio;
- entrenamiento;
- énfasis operacional;
- actividades guiadas;
- modo de práctica.

Criterio visual:

- puede usar acentos cálidos;
- debe mantener sobriedad institucional;
- no debe parecer una interfaz táctica, policial o de emergencia real.

## Reglas de layout

La interfaz debe soportar:

- Phone Vertical
- Tablet Vertical
- Desktop Horizontal
- Desktop HD
- Phone Horizontal
- Tablet Horizontal
- Desktop Vertical

El diseño principal de escritorio debe considerar una composición máxima de 1920x1080 px, centrada en pantallas mayores.

## Estados visuales

- enabled: visible y funcional.
- disabled: visible, pero no interactivo.
- coming_soon: visible como próxima funcionalidad.
- hidden: no visible.
- draft: visible solo para admin.
- archived: retirado, conservado en historial.

## Botón Notas

El botón flotante debe llamarse siempre:

Notas

No debe llamarse Bitácora en la interfaz flotante.

## Criterio legal-operacional

La plataforma es formativa. No debe registrar datos reales de víctimas, PSC, domicilios, teléfonos, coordenadas, folios, causas ni antecedentes identificables.

Todo diseño debe reforzar el uso educativo, la minimización de datos y la trazabilidad formativa.
